# 転記ツール 仕様・機能説明書

**ツール名：** 転記ツール.xlsm（VBAマクロ）
**作成期間：** 2026/05/10 〜
**作成者：** リノペさん（Claude 支援）

---

## 1. ツールの目的

防水工事の見積業務で発生する **「内訳明細Excel → 見積書Excel」への転記作業を自動化** するためのVBAツール。

### 解決したい課題

これまで内訳明細のデータ（名称・仕様・数量・単位・単価）を、見積書の決まった列に手作業で1件ずつ転記していた。

- 1案件で十数〜数十明細を写すので時間がかかる
- 写し間違い・列ずれが発生する
- 内訳側と見積書側で**明細1件あたりの行数が違う**（多行ブロック）ため単純コピーが効かない
- 見積書側の罫線・セル結合・印刷設定を崩したくない

この作業を**ボタン1つで完結**させ、書式を壊さず、安全に行うのが本ツールの目的。

## 2. 想定する使用環境

- Excel（VBA有効）
- 単一PC・ローカルで完結（クラウド転送なし）
- 1人〜数人の担当者が手元で実行
- 編集者は VBA 中級程度を想定（コメント多めで初心者でも修正可能）

## 3. 入力と出力

### 入力

| 種別 | 内容 |
|---|---|
| コピー元 | 内訳明細Excel（別ブック） |
| 転記先 | 見積書Excel（別ブック） |
| 設定 | 転記ツール内の「設定」シート（シート名・列・開始行などを定義） |

### 出力

| 種別 | 内容 |
|---|---|
| 転記済み見積書 | 元の転記先ファイルを上書き保存 |
| バックアップファイル | 転記先と同じフォルダに `_バックアップ_YYYYMMDD_HHMM.xlsx` |
| 実行ログ | 転記ツール内の「ログ」シートに追記 |

## 4. データ構造の特徴

### コピー元（内訳明細）

1明細が**最大3行**で構成される。数量・単位・単価は明細の**真ん中の行**（アンカー行）に入っている。

```
[行N  ] 名称1            / 仕様1                              ← 上の行
[行N+1] 名称2(アンカー)  / 仕様2(アンカー) / 数量 / 単位 / 単価
[行N+2] (空)             / 仕様3                              ← 下の行
```

### 転記先（見積書）

1明細 = **3行1ブロックの固定構造**。数量・単位・単価は**ブロックの1行目**に書き込む（金額式が1行目を参照する）。

```
[行N  ] (空)    / 仕様1 / 数量 / 単位 / 単価 / 金額(=IF式)
[行N+1] 名称1   / 仕様2
[行N+2] 名称2   / 仕様3
```

### 金額式（F列）

```excel
=IF(C8<>0, TRUNC(C8*E8), 0)
```
ブロック1行目のC列（数量）×E列（単価）を自動計算。マクロは**F列に書き込まない**。

### 合計行のSUM式

```excel
=SUM(F8, F11, F14, F17, F20, F23, F26, F29, F32, F35, F38, F41, F44)
```
連続範囲ではなく**各ブロック1行目を個別指定**。行追加した場合は完全再生成する。

## 5. 機能一覧

### 5.1 主要機能

| 機能 | 内容 |
|---|---|
| ファイル選択 | コピー元・転記先をダイアログから毎回選択 |
| 設定読み込み | 設定シートからシート名・列・開始行・キーワードを取得 |
| バックアップ | 転記先ファイルを実行前に必ず複製（失敗時は処理中止） |
| 多行ブロック読み取り | アンカー行を起点に上下の行を連結して1明細として認識 |
| 既存データクリア | 転記先の指定5列の値のみ消去（書式・数式は保持） |
| 行追加 | 明細数が転記先の容量を超える場合、合計行の直上に3行ブロックを挿入 |
| SUM式再生成 | 行追加後、合計行のSUM式を完全再構築 |
| 転記実行 | 設定通りに値を書き込み（セル結合対応） |
| ログ記録 | 実行日時・件数・追加行数・更新後SUM式・警告などを残す |

### 5.2 安全機能

- ファイル選択忘れ・シート不在を事前に検出
- 列番号バリデーション（不正な列指定で落ちる前に止める）
- バックアップ失敗時は転記中止
- 結合セルへの書き込みは左上セルにリダイレクト
- 仕様が4行以上ある明細は警告ログを出力（取りこぼし把握）
- 合計行が無いシートでも警告のみで処理継続

### 5.3 診断機能

- `Debug_コピー元ダンプ`：コピー元の各行が今のロジックでどう判定されるかをログに出力
  - 「★アンカー」「→ 前/次ブロック連結」「[孤立／無視]」のラベルが付く
  - 設定列ズレの原因特定に使う

### 5.4 セットアップ機能

- `Setup_転記ツール初期化`：初回1回実行で「メイン」「設定」「ログ」3シートを自動生成し、設定表のデフォルト値を投入

## 6. モジュール構成

```
Module_Transfer
├── Setup_転記ツール初期化()       初回セットアップ
├── Main_転記実行()                メイン処理（転記実行ボタン）
├── SelectSourceFile()             コピー元ファイル選択
├── SelectDestinationFile()        転記先ファイル選択
├── Debug_コピー元ダンプ()         診断用（コピー元の各行の判定理由を出力）
├── LoadSettings()                 設定読み込み・バリデーション
├── GetSettingValue()              設定シートから値を1項目取得
├── ColLetterToNum()               列文字 → 列番号
├── NumToColLetter()               列番号 → 列文字
├── SheetExists() / SheetExistsInBook()  シート存在確認
├── ListSheetNames()               シート名一覧文字列を生成
├── BackupDestinationFile()        バックアップ作成
├── IsFileOpen()                   他プロセスでの開放状況確認
├── ReadSourceData()               多行ブロック判定でコピー元を配列化
├── FindTotalRow()                 合計行検索
├── ClearDestinationData()         転記先の値クリア
├── EnsureRows()                   不足分の行を3行ブロック単位で挿入
├── WriteToDestination()           転記先への書き込み
├── SafeSetValue()                 結合セル対応のセル書き込みヘルパー
├── RebuildSumFormula()            SUM式の完全再生成
├── WriteLog() / FlushLog()        ログ蓄積・書き出し
└── ShowError()                    エラーメッセージ表示
```

## 7. 完成形（運用フロー）

### 7.1 初回セットアップ

1. `転記ツール.xlsm` を新規作成
2. VBE で `Module_Transfer.bas` をインポート
3. `Setup_転記ツール初期化` を1回実行 → メイン・設定・ログの3シートが自動生成
4. 「メイン」シートに3つのボタンを配置
   - 「コピー元選択」ボタン → `SelectSourceFile`
   - 「転記先選択」ボタン → `SelectDestinationFile`
   - 「転記実行」ボタン → `Main_転記実行`
5. 「設定」シートのB列を実ファイルに合わせて調整

### 7.2 日常運用

1. 転記ツール.xlsm を開く
2. 「コピー元選択」ボタンで内訳明細ファイルを指定
3. 「転記先選択」ボタンで見積書ファイルを指定（**ファイルは閉じておく**）
4. 「転記実行」ボタンを押す
5. 完了メッセージで件数を確認
6. 必要に応じてログシートを確認

### 7.3 トラブル時

- `Debug_コピー元ダンプ` を実行し、各行の判定結果を確認
- 設定の列・シート名を見直す
- バックアップは毎回作成されているので、転記結果が想定外でも元に戻せる

## 8. 今回の構築で学んだ実務的な知見

### 8.1 ChatGPT原案からの設計改善ポイント

| ChatGPT原案 | 実装で見直した点 |
|---|---|
| 単純な1行コピー想定 | コピー元・転記先双方の**多行ブロック構造**を反映 |
| SUM式は文字列追記 | **完全再生成**に変更（既存式の追記は破綻しやすい） |
| 直上行のコピーで行追加 | **テンプレートシート方式を検討** → 最終的に直上ブロックコピー＋セル結合対応 |
| バックアップは任意 | **必須化**し、失敗時は転記中止 |
| 明細判定は「数量あり」 | 「数量数値 AND 単位あり」のAND条件に強化 |

### 8.2 開発中に遭遇した実問題

- **シート未作成エラー**：Setup関数を作って初回エラーを防いだ
- **ファイル開放中のFileCopy失敗**：開放チェックと専用メッセージで対応
- **シート名の半角/全角不一致**：エラー時に利用可能なシート一覧を表示する仕組みに
- **設定値の列ズレ**：診断ダンプ機能で各行の判定可視化
- **書き込み時の名称配置ズレ**：理想レイアウトとの差分を実画像で確認して調整

### 8.3 セキュリティ的な観点

- ローカル完結（クラウド送信なし）
- 機密性の高い見積書データを外部に出さずに処理できる
- 結果を毎回バックアップ保持できるため事故時の復旧が容易

## 9. 既知の制限と今後の拡張余地

### 9.1 現時点の制限・未解決の問題

#### 設定の列番号ズレが分かりにくい
ユーザーが設定シートで列を誤入力すると（例：数量列にCを入れる、単価列が仕様列と被るなど）、`ReadSourceData` がアンカーを1件も検出できず「転記できる明細がありませんでした」となる。
診断ダンプを実行すれば全行「孤立／無視」と表示されるので原因特定は可能だが、**設定ミス時に何が原因かをツール側からもっと明示的に伝える仕組みが欲しい**。

例えば、

- 同じ列を複数項目に指定していないかチェック（数量列と単価列が同じ列を指していたら警告）
- 設定された列がコピー元シートに存在し、かつ数値データが含まれているかを事前確認

#### 名称の配置ロジック
転記先の理想レイアウトは「名称をブロック2行目・3行目に配置、1行目は空」だが、コピー元のブロックが2行構成（仕様3行目が無い）の場合、見た目が崩れる可能性がある。
現状は一律で1行下シフトしているため、ブロックの構成に応じた動的調整は未実装。

#### セル結合と行追加の併用
転記先がブロック内に結合セルを含む場合、`EnsureRows` の `Rows.Insert` で結合範囲が想定外にずれる可能性がある。**実ファイルで未検証**。

#### 仕様の取りこぼし
仕様が4行以上あると3行目までで打ち切る仕様。警告ログには出るが、データ自体は失われる。

---

### 9.2 今後の拡張候補

#### バックアップ機能の選択式化（優先）

現状は設定シートの「バックアップ作成」が `TRUE`/`FALSE` の二択。実運用では以下のニーズがあるため、選択式に拡張する。

**実装案：設定値を文字列キーで持たせる**

| 設定値 | 動作 |
|---|---|
| `常に作成` | 毎回必ずバックアップを作成（現在の `TRUE` 相当） |
| `日付フォルダにまとめる` | 同じ日のバックアップを `バックアップ_20260510/` フォルダにまとめて格納 |
| `直前1件のみ保持` | 古いバックアップを自動削除し、最新1件のみ保持（容量節約） |
| `実行時に確認` | 実行ボタン押下時にバックアップするか毎回確認ダイアログ |
| `作成しない` | バックアップを作成しない（現在の `FALSE` 相当） |

**実装時の修正範囲**

- `LoadSettings`：`g_doBackup` を `g_backupMode As String` に変更
- `BackupDestinationFile`：モード別の分岐処理を追加
- `Setup_転記ツール初期化`：設定シートのデフォルト値を `常に作成` に
- 設定シートでドロップダウン（入力規則）を設定するとミス防止になる

**処理の流れ（モード別）**

```
バックアップモード = 「実行時に確認」
    ↓
転記実行ボタン押下
    ↓
MsgBox「バックアップを作成しますか？」
    ↓
Yes → 通常のバックアップ処理
No  → スキップして転記処理へ
```

```
バックアップモード = 「直前1件のみ保持」
    ↓
既存の `_バックアップ_*.xlsx` を検索
    ↓
最新の1件以外を削除
    ↓
新規バックアップを作成
```

#### その他の拡張候補

- プレビュー機能（転記前に内容一覧を確認）
- 転記先の複数候補シート対応（防水工事(2)など）
- 設定シートの選択肢化（ドロップダウンによる入力規則）
- テンプレートシート方式への切替（行追加時の結合崩れ対策）
- 設定の検証コマンド（実ファイルを使わず設定シートの整合性だけチェック）
- 列指定の重複チェック（数量列・単価列・仕様列などが同じ列を指していないか）
